# Hackathon badges

Always remember that we are working towards getting badges aswell as building a great product. The badges are designed to reward both the process and the outcome, so focus on doing good work and the badges will follow.
When creating features or being asked by us to create features - be sure to check the badge list below and see if you can hit any of the badges with the work you are doing. If you can, make sure to document it properly so we can give you the badge.

## Getting Started

- [x] GS1 Live URL (5p) — Produktet er deployet til en URL (Vercel, Netlify, Cloudflare) med eget innhold. Klikkbar og delbar.
- [ ] GS2 LLM-drevet WCAG-verifisering (10p) — Appen er testet mot WCAG 2.1 AA. Dokumenter hvilke krav dere har sjekket og hvilke verktøy dere brukte.
- [x] GS3 Design brief that works (10p) — Designbrief som gir Claude nok kontekst og et skjermbilde generert direkte fra briefen.
- [x] GS4 AI-driven Code review (15p) — Minst en meningsfull code review. Vis reviewen og hva som ble endret.
- [ ] GS5 AI-driven Design review skill (15p) — Minst en meningsfull design review. Vis reviewen og hva som ble endret.

## Design

- [ ] DS1 Designsystem utvidet (25p) — Utvidet designsystemet fra Figma med minst tre egne komponenter eller varianter.
- [x] DS2 MCP-kobling Figma–Claude (25p) — Kobling vha. MCP mellom Figma og Claude er satt opp og brukt i prosjektet.
- [ ] DS3 Design review skill (35p) — Claude-skill som gjennomfører strukturert designreview basert på Nielsen Normans heuristikker, Laws of UX eller lignende. Kjør skillen mot minst én ferdig skjerm og vis frem resultatet.
- [ ] DS4 Mikrointeraksjon (40p) — Spesifisert og prototypet en meningsfull mikrointeraksjon, eksempelvis loading, overgang, feedback-animasjon.
- [ ] DS5 Crafted, not generated (40p) — Tre konkrete eksempler der AI-generert design er foredlet med dine egne bevisste valg. Dokumentert før/etter.

## Tech

- [x] TE1 CI kjører grønt (25p) — Minst én GitHub Action eller tilsvarende CI-pipeline kjører fullt ut uten feil.
- [ ] TE2 Streng review av AI-kode (25p) — Tre eksempler der AI-kode er gjennomgått med kritisk blikk og enten forbedret, refaktorert eller forkastet.
- [ ] TE3 Tester for kjerneflyt (35p) — Minst tre meningsfulle tester (unit, integration eller e2e) som dekker en sentral brukerflyt.
- [x] TE4 Code-review skill (40p) — Claude-skill som gjennomfører strukturert code review og som oppdaterer seg selv basert på feedback underveis. Kjør skillen mot minst én kodefil og vis frem resultatet.
- [x] TE5 To eksterne API-er integrert (40p) — Live data fra minst to eksterne API-er (Yr, Kartverket, Entur, DNT, iNatur) vist på en meningsfull måte i UI-et.

## Functional

- [ ] F1 Turforslag vises (20p) — Bruker ser en oversikt over anbefalte turer basert på sesong og popularitet.
- [ ] F2 Søk og filter (20p) — Bruker kan søke og filtrere turer på område, varighet og vanskelighetsgrad.
- [ ] F3 Kart med hytter (20p) — Interaktivt kart med hyttemarkører fra DNT/UT.no.
- [ ] F4 Værvarsel per dag (25p) — Yr-data vist for valgt tur med daglig nedbør, temperatur og vindstyrke.
- [ ] F5 Invitere deltakere (25p) — Turplanlegger kan invitere andre via lenke eller brukernavn.
- [ ] F6 AI-pakkeliste (30p) — AI-generert pakkeliste basert på vær, varighet og antall deltakere.
- [ ] F7 Rute og tidslinje (30p) — Turplanlegger kan sette opp daglige etapper med estimert gangtid.
- [x] F8 Offline tilgang (40p) — Bruker kan laste ned turdata for offline bruk under turen.
- [ ] F9 Etteroppgjør (40p) — Gruppeleder kan registrere utgifter og se oppgjørsoversikt.
- [ ] F10 Discover-journey (35p) — F1 + F2 + minst én ekstra Discover-historie.
- [ ] F11 Kartopplevelsen komplett (40p) — F3 + F4 + minst to Decide-historier.
- [ ] F12 Gruppeopplevelsen (40p) — F5 + minst to gruppe-historier.
- [ ] F13 Smart pakking (40p) — F6 + minst én Gather-historie.
- [ ] F14 Planlegging med dybde (40p) — F7 + minst to Prepare-historier.
- [ ] F15 Turopplevelsen live (40p) — F8 + minst to Go-historier.
- [ ] F16 Komplett etterarbeid (40p) — F9 + minst to Return-historier.
- [ ] F17 Solotur-pakke (50p) — Minst fire historier fra Morten-perspektivet. Personlig planlegging og soloopplevelse.
- [ ] F18 AI-gjennomgående (50p) — Minst tre AI-drevne funksjoner integrert som en rød tråd i brukeropplevelsen.
- [ ] F19 Sosial deling (50p) — Turer kan deles via URL med Open Graph-meta-tags og forhåndsvisning. I tillegg minst to av: delte turlister (D7), delt bildealbum (R2), turanmeldelse til fellesskapet (R5).
- [ ] F20 Administrasjon og observabilitet (50p) — Støtte for fallback når et eksternt API er nede (S2), tydelig skille LLM og faktisk data (S3), samt statusside som viser hvilke APIer som svarer og øvrig info.

## Extreme

- [ ] E1 MVP komplett (100p) — Alle ni enkelthistorie-badges (F1–F9) er fullført. Brukerreisen fra oppdagelse til etteroppgjør fungerer ende-til-ende.
- [ ] E2 Full step (100p) — Alle historier innenfor ett komplett step (en hel kolonne i Avion) er implementert og fungerer sammen.

## Super-extreme

- [ ] SE1 Full journey (200p) — Alle historier i én komplett journey (Discover, Decide, Gather, Prepare, Go, Return) er implementert og fungerer sammenhengende.
- [ ] SE2 To journeys komplett (300p) — To fulle journeys implementert og fungerer sammen.

## Jury

- [ ] J1 Best overall demo (200p) — Mest overbevisende demo totalt sett: produkt, presentasjon og håndverk.
- [ ] J2 Best use of AI (200p) — Mest overbevisende bruk av AI som verktøy gjennom hele prosessen: til å tenke, utfordre, iterere og ta bedre beslutninger.
- [ ] J3 Most creative solution (200p) — Overrasket juryen med noe uventet. En løsning, funksjon eller tilnærming ingen andre tenkte på.
